# 🐉 数码宝贝虚拟世界 - 设计文档 (Design Document)

> 最后更新: 2026-07-07 · 维护: 星航

## 1. 项目愿景

构建一个**涌现式数字生命世界**。用户以**观察者/导演**身份进入,观看数码兽们在 2D 插画世界里自主生活、战斗、进化、社交。

**核心特征**:
- 不是聊天机器人,是一个会自己"活"起来的世界
- 借鉴 Stanford Generative Agents (2023) 的多智能体架构
- 复用数码宝贝动画的世界观与角色体系
- 长期运行,世界状态持久化

## 2. 世界设定 (数码世界 / Digital World)

### 2.1 顶层结构
- **数码世界** (Digital World) — 故事发生地,与现实平行
- **现实世界** (Real World) — 入口,被选召的孩子们的故乡
- **黑暗领域** (Dark Area) — 反派栖居地

### 2.2 主要地区

| 地区 | 灵感 | 主要数码兽 |
|------|------|----------|
| **无限山** Infinity Mountain | 数码世界中心,创世神栖息地 | 创世兽、究极体 |
| **文件岛** File Island | 主角们初到的小岛 | 幼年期~成熟期 |
| **沙拔大陆** Server Continent | 大陆,城市与荒野并存 | 各种族 |
| **黑暗龙卷山** Dark Tower | 反派据点 | 黑暗四大天王 |

### 2.3 关键地点
- **启程海滩** (Beach of Departure) — 数码兽与被选召的孩子相遇的地方
- **进化神殿** (Evolution Shrine) — 数码兽进化的场所
- **神圣计划祭坛** (Crests Altar) — 徽章激活处
- **被选召的孩子宿舍** (Chosen Children HQ) — 用户可选入口
- **奥加兽的商店** (Ogremon's Shop) — 道具交易点

### 2.4 时间系统
- 现实 1 秒 = 世界 1 分钟 (默认,可调)
- 1 个世界日 = 现实 24 分钟
- 数码兽需要睡眠、觅食、社交,作息规律

## 3. 智能体架构 (Agent Architecture)

参考 **Stanford Generative Agents** (Park et al., 2023)。每个 Agent 是一个 LLM 驱动的循环。

### 3.1 核心循环 (DigimonAgent Loop)

```
        ┌────── Observe ──────┐
        │   (环境/其他 agent  │
        │   /用户事件)        │
        └──────┬──────────────┘
               ▼
        ┌────── Memory ───────┐
        │  写入记忆流         │
        │  (带重要性评分 1-10)│
        └──────┬──────────────┘
               ▼
        ┌────── Reflect ──────┐
        │  阈值触发           │
        │  生成高级抽象反思   │
        └──────┬──────────────┘
               ▼
        ┌────── Plan ─────────┐
        │  生成下一段计划     │
        │  (移动/对话/战斗/   │
        │   觅食/休息)        │
        └──────┬──────────────┘
               ▼
        ┌────── Act ──────────┐
        │  执行计划           │
        │  产生世界事件       │
        └──────┬──────────────┘
               └──────→ (回到 Observe)
```

### 3.2 关键设计决策

| 项 | 决策 | 理由 |
|----|------|------|
| **模型分层** | opus 4.8 反思/重要决策;haiku 4.5 大量移动/观察 | 控制成本 |
| **反思频率** | 每累计 100 条记忆 或 每 24 世界小时 | 避免过频反思 |
| **记忆检索** | 时序近因 + 重要性 + 关联性 3 维评分 | 借鉴 Stanford |
| **计划粒度** | 高层计划(天级) → 子计划(小时级) → 行动(分钟级) | 层次化 |
| **持久化** | 记忆流 + 反思树存 SQLite | 跨重启 |

### 3.3 Agent 类型

- **数码兽 Agent** (DigimonAgent) — 主要角色,有进化体系
- **NPC Agent** — 普通居民,无进化
- **被选召的孩子 Agent** (ChosenChildAgent) — 预备,等用户进入世界时使用
- **观察者** (Observer) — 不是 agent,是 UI 视角

## 4. 数码兽机制 (Digimon System)

### 4.1 进化链 (5 阶段)

```
幼年期 (Baby I)      ← 刚孵化
   ↓
成长期 (Baby II / In-Training)
   ↓
成熟期 (Rookie)      ← 玩家可互动起点
   ↓
完全体 (Champion)
   ↓
究极体 (Mega / Ultimate)
```

(原作中还有"装甲体"和"超究极体",本项目预留接口)

### 4.2 进化触发条件

进化的本质是**LLM 在"剧情高潮"时决定触发**,由以下条件综合:

- 战斗胜利累计数
- 与训练师 / 被选召的孩子的羁绊值 (0-100)
- 关键剧情事件(被感化、顿悟、危机时刻)
- 数码兽自身的"意愿"(LLM 评估)

### 4.3 属性克制 (Data/Vaccine/Virus)

| 属性 | 克制 | 被克制 |
|------|------|--------|
| 疫苗种 (Vaccine) | 病毒种 | 数据种 |
| 数据种 (Data)   | 疫苗种 | 病毒种 |
| 病毒种 (Virus)  | 数据种 | 疫苗种 |
| 自由种 (Free)   | 无克制 | 无克制 |

## 5. 战斗系统 (Battle System)

### 5.1 基本规则
- 回合制,每个数码兽每回合行动一次
- 行动选项: **攻击 / 防御 / 技能 / 逃跑 / 道具**
- 技能消耗能量(EP),能量随回合恢复
- HP 归零 = 战斗失败(原作中数码兽不会真正"死",只是"消失回数码世界")

### 5.2 战斗 AI
战斗决策由 **LLM** 驱动,从以下信息推理:
- 数码兽自身性格(勇气/友情/爱心/知识/诚实/纯真/希望/光明)
- 记忆中的对手行为模式
- 当前属性克制
- HP/EP 状态
- 剧情张力(LLM 自评"现在是否适合爆发/撤退")

### 5.3 战斗视角
- 观察者模式: 看,不打
- 战斗发生由世界事件触发(野数码兽、剧情冲突、用户导演注入)

## 6. 观察者 / 导演模式 (Observer / Director Mode)

### 6.1 观察者 UI
- **全景地图**: 2D 插画风,可缩放,显示所有 agent
- **选中某 agent**: 弹出侧栏,显示其状态、记忆摘要、当前计划
- **时间控制**: 暂停 / 1x / 2x / 5x / 10x / 60x
- **统计面板**: 数码兽总数、战斗数、进化次数、剧情事件数

### 6.2 导演能力
- 注入世界事件: "沙拔大陆突然出现异常波动"
- 创造新数码兽: "在文件岛生成一只受伤的亚古兽"
- 修改世界状态: 调整时间、天气、地点属性
- 加速某个 agent: 让某数码兽"觉醒"特别剧情

### 6.3 不参与原则
- 观察者**不进入战斗**
- 观察者**不对话**(但可让两个 agent 对话)
- 观察者**不持有数码兽**(除非后续开启"被选召的孩子"模式)

## 7. 技术栈选型

| 层 | 选择 | 理由 |
|----|------|------|
| **后端语言** | Python 3.11+ | LLM 生态成熟,异步原生 |
| **Web 框架** | FastAPI | 异步、自动 OpenAPI 文档 |
| **Agent 编排** | LangGraph 或自研 | LangGraph 适合有状态循环 |
| **LLM 客户端** | httpx 直连中转 | 简单可控,opus 4.8 + haiku 4.5 |
| **前端** | HTML5 Canvas + PixiJS | 2D 插画,比纯 Canvas 简单 |
| **存储** | SQLite (起步) | 轻量,后续可换 Postgres |
| **向量检索** | 暂用关键词 + 时序,后期可加 sqlite-vss | 避免初期复杂度 |
| **任务队列** | 暂用 asyncio,后期可上 Celery/RQ | 后期 |

## 8. 风险与挑战

| 风险 | 缓解 |
|------|------|
| **LLM 成本**: 大量 agent 自主决策 token 开销巨大 | 分层模型(opus 反思, haiku 移动) + 反思频率限制 |
| **涌现失控**: agent 之间可能产生不可预测剧情 | 导演面板有"重置该世界"功能,反思可触发"重启" |
| **长期一致性**: 记忆流过大时检索效率下降 | 反思压缩旧记忆,重要性评分排序,定期归档 |
| **数码宝贝 IP**: 万代版权 | 纯粉丝/研究项目,非商业,显著标注 IP 归属 |
| **API 中转不稳定**: 第三方服务随时可能挂 | LLM 客户端支持 fallback 模型,关键路径 try-retry |
| **冷启动问题**: 一开始世界空荡,不好看 | 开局注入 5-10 只数码兽 + 一段"序章"剧情剧本 |
| **评估困难**: 怎么判断世界"活"起来了? | 定性: 留观一段故事,数码兽是否自然对话/战斗/进化 |

## 9. 后续可扩展

- 联网: 数码兽可以"感知"现实世界新闻(注入到记忆流)
- 用户进入: 开启"被选召的孩子"模式,用户成为训练师
- 跨世界: 多个数码世界实例通过"数码之门"互通
- 模因: 数码兽可以"教会"彼此技能,技能可以"流传"
- 季节系统: 春/夏/秋/冬,影响数码兽习性
- 数字死亡: 究极体被击败后降级(原作设定)
- 数码兽的"梦想": 每个数码兽有长期目标(LLM 生成)
